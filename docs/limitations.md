# 能力边界与下一步

> 本项目仅作为个人项目作品集展示。公开版本提供可运行的合成数据 Demo；完整 AI 链路需要单独配置外部服务与未公开数据。

## 当前边界

- 公网演示是引导式合成交互，不支持任意问题的在线真实生成。
- 公网定位只在浏览器内近似匹配到内置演示城市，不上传坐标；本地完整模式才调用高德逆地理编码。
- Query State 当前为进程内存储，多 worker 部署需迁移到 Redis 等共享存储。
- Dify blocking 链路 P95 偏高，需要流式响应和节点级性能优化。
- 早期 Extraction 缓存没有直接输出新版原子 Claim，当前存在兼容迁移层。
- 高德路线估算不等于完整铁路、轮渡班次或票务信息。

## 下一步

1. 给 Extraction Schema 增加版本并重新抽取公开授权数据。
2. 建立人工 Top3 相关性标注，补充 NDCG 与 pairwise 排序评测。
3. 将 Query State 外置并增加鉴权、限流和可观测性。
4. 优化 Dify 节点延迟，改为流式进度和结果返回。
5. 在真实流量达到样本要求后再判断定位实验效果。
