# 产品案例：从模糊感觉到可信目的地

## 问题

用户在灵感阶段往往没有目的地，只能表达“想安静一点”“想看海”“不想太商业化”。传统 OTA 的目的地搜索、销量排序和库存模型无法直接处理这种意图。

## 产品目标

帮助用户用低负担表达感觉，在预算、天数和交通可执行的前提下，找到愿意进一步考虑的目的地。

北极星指标：

```text
有效灵感转化率
= 有“想去/收藏/明确正反馈”的需求会话
÷ 已提交旅行推荐需求的会话
```

## MVP 边界

包含：目的地发现、一次渐进澄清、硬约束过滤、感觉排序、证据理由、透明降级和推荐反馈。

不包含：酒店机票预订、账户社交、多模态内容生产、完整铁路时刻表和自动执行型 Agent。

## 核心决策

1. 出发地是唯一阻断槽位；预算和天数缺失时允许带假设推荐。
2. Query Plan 分离硬约束、软偏好、排除项和证据需求。
3. 检索不直接决定最终顺序，业务后端负责过滤、排序和证据准入。
4. 推荐理由必须绑定 Claim；事实字段不得由 LLM 改写。
5. 空结果是合法产品状态，不用幻觉填满三张卡片。
